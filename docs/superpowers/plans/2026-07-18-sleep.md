# Sleep Tracker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Sleep timer with pause/resume (pause rows), editable history with pause deletion, 7-day chart, Home/Track integration.

**Architecture:** Mirrors the feeding feature 1:1 where possible. `sleeps` + `sleep_pauses` rows are the timer state; state (`running`/`paused`/`ended`) is derived, never stored. Pure `sleepMath.ts` computes everything from rows + injected `nowMs`. `useActiveSleep` polls 10 s and returns the sleep WITH its pauses (single query, Supabase nested select). Screens copy feeding's structures (ActiveFeedCard → ActiveSleepCard, feeding/index → sleep/index, feeding/[id] → sleep/[id] with dirty-field gating).

**Tech Stack:** Existing. No new dependencies.

### Task 2: sleepMath (TDD)

**Files:**

- Create: `src/features/sleep/sleepMath.ts` + `src/features/sleep/sleepMath.test.ts`

**Interfaces:**

- Produces: `pauseSeconds(pauses: SleepPause[], nowMs): number`; `effectiveSleepSeconds(sleep: Sleep, pauses: SleepPause[], nowMs): number`; `sleepState(sleep: Sleep, pauses: SleepPause[]): 'running' | 'paused' | 'ended'`; `sleepSummary(sleep: Sleep, pauses: SleepPause[]): string`; `dailySleepTotals(sleeps: SleepWithPauses[], days: number, todayIsoStr: string): { dateIso: string; count: number; totalSeconds: number }[]`.

Contract details:

- `pauseSeconds`: Σ (ended|now − started), floor per pause, clamp ≥ 0 each.
- `effectiveSleepSeconds`: (ended|now − started) − pauseSeconds, clamp ≥ 0.
- `sleepState`: ended_at set → 'ended'; else any pause with ended_at null → 'paused'; else 'running'.
- `sleepSummary`: `formatDuration(effective)` + (pauses.length ? ` · ${n} pause${n===1?'':'s'}` : '') — for ENDED sleeps callers pass nowMs irrelevant; signature takes nowMs? NO — summary is for ended sleeps in lists; compute with `nowMs = Date.parse(sleep.ended_at ?? sleep.started_at)` INTERNALLY? Cleaner: `sleepSummary(sleep, pauses, nowMs)` — explicit, pure, same as other fns. Use 3-arg form.
- `dailySleepTotals`: mirror feedMath's `dailyTotals` local-date bucketing exactly (reuse `localDateIso` — import it from feedMath, or move `localDateIso`/`localDateParts` into `@/lib/dates` and update feedMath imports; prefer the move, it's 2 small functions and dates.ts is their natural home). Bucket by started_at local date; totalSeconds = effective sleep computed with nowMs (running sleep counts progress so far); count per bucket.

TDD: tests first (cover running/paused/ended elapsed, open-pause counting, clamps with fractional-second deltas — pin floor vs round with a 90.4 s delta, singular/plural pause label, empty middle day via `localDateIso`-derived expectations), RED, implement, GREEN.

### Task 3: Sleep hooks

**Files:**

- Create: `src/features/sleep/hooks.ts`

**Interfaces (mirror feeding/hooks.ts structure, `['sleeps']` key namespace):**

- `useSleeps(babyId)` — `SleepWithPauses[]`, `.select('*, sleep_pauses(*)')`, order started_at desc.
- `useActiveSleep(babyId)` — `SleepWithPauses | null`, same nested select, `.is('ended_at', null)`, `refetchInterval: 10_000`, limit 1 maybeSingle.
- `useStartSleep()` — `{ babyId }`: insert sleep with started_at = now ISO, created_by stamped.
- `usePauseSleep()` — `{ sleepId }`: insert sleep_pauses row (started_at now).
- `useResumeSleep()` — `{ pauseId }`: update pause set ended_at now.
- `useStopSleep()` — `(sleep: SleepWithPauses)`: if open pause exists, set its ended_at = now; then set sleep.ended_at = now. Two awaited updates, pause first.
- `useUpdateSleep()` — `Partial<Pick<Sleep,'started_at'|'ended_at'|'note'>> & { id }`.
- `useDeleteSleep()` — id (cascade removes pauses).
- `useDeletePause()` — pauseId.
  All mutations invalidate `['sleeps']` prefix.
